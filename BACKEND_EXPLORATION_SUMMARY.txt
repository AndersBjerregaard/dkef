1. FILE STORAGE (MinIO/S3)
================================================================================

ARCHITECTURE:
  - MinIO (S3-compatible object storage)
  - 3 separate buckets: "events", "news", "general-assemblies"
  - Server: src/dkef-api/Services/MinioBucketService.cs
  - Interface: IBucketService.cs

PRESIGNED URLS:
  ✓ Generated via BucketController endpoints (/bucket/{type}/{id})
  ✓ PUT-only (upload), 1-hour expiry
  ✓ HMAC-SHA256 signed by MinIO
  ✓ Public-read bucket policy on creation
  ✓ HTTPS rewriting support for TLS-terminating proxies

IMAGE LIFECYCLE:
  ✓ Upload: Direct to MinIO via presigned URL (client-side)
  ✓ Storage: GUID as object name
  ✓ Update: Old image deleted on thumbnail change (fire-and-forget)
  ✓ Delete: Old image cleaned up when entity deleted (non-blocking)
  ✓ Cleanup: ImageCleanupService removes orphaned images

CONFIGURATION:
  - ConnectionStrings:Minio (public endpoint)
  - ConnectionStrings:MinioInternal (optional cluster endpoint)
  - Minio:PublicEndpoint (optional custom domain)
  - Minio:AccessKey, SecretKey, Secure

COMMON PATTERNS:
  ✓ All DTOs inherit PostObject (enforces Sanitize method)
  ✓ All domains inherit DomainClass (provides Guid Id)
  ✓ ThumbnailId (GUID string) maps to ThumbnailUrl (full URL) via AutoMapper
  ✓ All timestamps UTC, stored as "timestamp with time zone"
  ✓ String properties default to string.Empty (never null)
  * = Required, (opt) = Optional

5. FRONTEND-BACKEND INTEGRATION
================================================================================

FILE UPLOAD WORKFLOW:
  1. Frontend generates UUID (uuid v4)
  2. Requests presigned URL: GET /bucket/{type}/{uuid}
  3. Uploads file to MinIO: PUT {presignedUrl} with file
  4. Updates entity: POST/PUT /events with dto.thumbnailId=uuid

COMPONENTS:
  ✓ EditEventModal.vue          → Upload & update event
  ✓ EditNewsModal.vue           → Upload & update news
  ✓ EditGeneralAssemblyModal.vue → Upload & update GA

FEATURES:
  ✓ File input with accept="image/*"
  ✓ Empty file validation (size === 0)
  ✓ Error handling & user feedback
  ✓ Extract GUID from existing URL
  ✓ Optional file replacement

URL SERVICE (src/dkef-vue/src/services/urlservice.ts):
  getEventPresignedUrl(guid)          → /bucket/events/{guid}
  getNewsPresignedUrl(guid)           → /bucket/news/{guid}
  getGeneralAssemblyPresignedUrl(guid) → /bucket/general-assemblies/{guid}

7. DESIGN PATTERNS & ARCHITECTURE
================================================================================

REPOSITORY PATTERN:
  Base:  IRepository<T, Y> where T=domain, Y=DTO
  Methods: GetByIdAsync, GetMultipleAsync, CreateAsync, UpdateAsync, DeleteAsync
  Impls: EventsRepository, NewsRepository, GeneralAssemblyRepository

PAGINATION:
  Type: DomainCollection<T> { Collection: IEnumerable<T>, Total: int }
  Usage: Skip-based pagination with item count

SORTING:
  Attribute: [Sortable] on DateTime/CreatedAt/PublishedAt
  Service: QueryableService<T> generates dynamic LINQ expressions

MAPPER:
  Library: AutoMapper
  Config: Program.cs lines 226-251
  Key mapping: ThumbnailId → ThumbnailUrl (URL construction)

IMAGE CLEANUP:
  Pattern: Fire-and-forget async (non-blocking)
  Execution: ConfigureAwait(false) with no await
  On Update: Delete old image if changed
  On Delete: Delete associated image
  Guarantee: Never blocks entity operations
